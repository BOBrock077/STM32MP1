## 三、MP1 靜態 IP 設定檔

檔案位置：`/etc/systemd/network/10-eth0-static.network`
（`10-` 開頭優先序高於原本的 `/lib/systemd/network/50-wired.network`，因此覆蓋其 DHCP 設定）

```ini
[Match]
Name=eth0

[Network]
Address=100.192.0.50/24
```

套用方式（修改後）：
```sh
systemctl restart systemd-networkd
```

> 原始預設設定 `/lib/systemd/network/50-wired.network` 為 `DHCP=ipv4`；
> 因本線路為直連、無 DHCP 伺服器，故改用上述靜態設定。

## 四、連線方式

從電腦連到 MP1：

```sh
ssh root@100.192.0.50          # SSH（若服務已開）
ping 100.192.0.50              # 連通性測試
```

序列主控台（不需網路）：
```powershell
python run_cmd_mp1.py "ifconfig eth0"   # 送單一指令並取回應
python monitor_mp1.py                   # 即時監控序列輸出
```

## 五、排查紀錄（為何原本拿不到 IP）

1. **現象**：eth0 設成 DHCP（`DHCP=ipv4`），線有接、有收封包，但一直 `carrier (configuring)`，拿不到 IPv4。
2. **測試 192.168.0.x**：設臨時靜態 IP ping 電腦舊位址 `192.168.0.33`／閘道 `192.168.0.99` → `Destination Host Unreachable`（ARP 失敗）→ 不在同一網段。
3. **電腦改用 `100.192.0.23`** 後，設臨時靜態 `100.192.0.50` ping → **通**，且 ARP 顯示對端 MAC `a0:ad:9f:43:d7:39` 即電腦網卡 → 確認 **PC 與 MP1 為直連**。
4. **根因**：直連線上沒有 DHCP 伺服器，故 MP1 永遠要不到 DHCP 位址。
5. **解法**：改設**靜態 IP `100.192.0.50/24`**（永久），雙向連通正常。
